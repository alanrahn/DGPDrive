DGP HTTP Web Services
NameDescriptionStatus
Beta Release 0.91The initial beta release of the DGP HTTP Web ServicesBeta

The HTTP web services are no longer used by the DGPDrive application and other applications in favor of the SOAP web services. This is mainly to enable the optional message encryption to be used when needed, but also for the SOAP one-way services used by the AutoWork subsystem. The HTTP web service remains unchanged, and can be used with TLS encryption as an alternative.

DGP SOAP Web Services
NameDescriptionStatus
Beta Release 0.91The initial alpha release of the DGP SOAP Web ServicesAlpha

The SOAP web services have been added to the DGPDrive system to include two types of functionality. First, they add built-in encryption of the API messages using a PGP style of hybrid cryptosystem (a combination of RSA asymmetric encryption and AES symmetric encryption) as an alternative to TLS. Second, they add one-way methods using the request/acknowledge pattern (which is only provided by the SOAP protocol) for multi-threaded parallel execution of the many iterations of various automated processes.

AutoWork Application
NameDescriptionStatus
Beta Release 0.91The initial beta release of the AutoWork applicationAlpha

The AutoWork subsystem requires a very reliable, scalable and performant mechanism for the multi-threaded parallel execution of many process iterations. The third version of this subsystem is being built using SOAP one-way web service methods (request/acknowledge pattern), which allows the AutoWork Windows service to return to being a simple scheduler application.
